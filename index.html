基礎說明

如何使用本指令包

每一份指令都是一塊「可複製的積木」。四個步驟，開始體驗把 AI 變成你的專屬助教。

1

找指令

翻到「指令速查表」或用目錄的年段跳轉，找到符合需求的指令編號。

2

複製貼上

點卡片中的「複製指令」，貼到你慣用的 AI 工具（ChatGPT、Gemini、Claude…）。

3

填入內容

把指令中以【】標註的地方，換成你的課文、單元或實際情境。

4

微調驗證

檢視 AI 產出，依班級程度微調，並務必確認內容正確、適切。

使用前的小提醒
·保護個資：勿輸入學生姓名、身分證號、地址等可辨識個資，可以用去識別化代稱、代碼描述。
·老師把關：AI 產出僅為草稿，正確性與適切性仍須教師審閱後再使用。
·鼓勵思考：把 AI 當成協作夥伴，師生應用時應善用工具，不依賴答案。
卡片說明

看懂每張指令卡片

每張卡片都標了兩個標籤：這是哪一種「指令類型」，以及它會幫你做出哪一種「產出」。先認識這兩組分類，就能依需求快速挑選。

兩種指令類型

新手建議先從「即用指令」上手，熟悉後再試「專屬助理」。

即用指令
＝ 複製貼上，馬上用

像借一張寫好的點菜單，貼進對話框、填上你的狀況就能快速產出內容。最適合單一任務、快速備課或者討論發想靈感、想要一份草稿的時候。

專屬助理
＝ 設定一次，長期幫你

像請一位固定助理，設定一次，之後每次找它都記得自己的角色、任務還有參考資料。適合會一直重複做的事（每單元教案設計、出考卷；特定風格圖片產生器、通知單產生器等）。在不同工具名字不同，例如 Gemini 叫「Gem」、ChatGPT 叫「GPT」等。

四種產出類型

卡片右上另標示這則指令會幫你做出什麼，方便你依需求快速挑選。

教學設計
課程流程、提問與評量的規劃建議。
互動教具
可操作的網頁小工具，如翻牌、隨機點名。
AI 生圖
請 AI 直接畫出教學插圖或佈置素材。
可印教材
一頁可直接列印的學習單、字卡或練習卡。
指令觀念

寫好一個指令的基本觀念

不必是工程師也能寫出好指令。把握以下的「要素」與「框架」，就能寫出清楚、好用的指令。

一、一個好指令的四個要素

角色 Persona
請 AI 扮演誰（如：國小自然老師）。
任務 Task
具體要它「做什麼」。
脈絡 Context
年段、程度、限制與背景資訊。
格式 Format
希望輸出的長相（表格、條列、字數）。

二、兩個好記的指令框架

PTCF
Persona 角色 · Task 任務 · Context 脈絡 · Format 格式
最通用的萬用框架，適合基礎快速對話或一次性的單一任務。
PARTS
Persona 角色 · Act 行動 · Recipient 受眾 · Theme 主題/風格 · Structure 結構 
架構更完整，適合更精確規劃，細節更完整的情境需求，善用 AI 高級推理能力。
選對工具

選擇適合的 AI 模型

同一則指令，用不同「等級」的模型，產出品質可能差很多。除了寫好指令，選對模型也是關鍵。

高階模型（進階／付費版）

推理更強、指令遵循度更高、產出更完整穩定。

適合：多步驟推理與運算、長文本與教材彙整、需要一次到位的文件／表格／簡報，以及互動工具、網頁、程式等較複雜的產出。

基礎模型（免費／輕量版）

回應快、隨手可用，日常任務多半綽綽有餘。

適合：改寫潤稿、發想點子、簡短問答、單一步驟的小任務。免費版常有用量上限，遇到複雜任務品質也較不穩定。
實用建議
重要或複雜的任務（尤其是要生成文件、工具或網頁），值得斟酌切換到高階模型；若免費版碰到額度上限，可改用其他工具、待額度恢復，或把任務拆小分次進行。各家工具的模型名稱與分級不同，以你使用的平台當下說明為準。

附錄二 · 進階應用

進階：Skill 架構（檔案代理）

前面的「即用指令」和「專屬助理」都在聊天視窗裡用。Skill 更進一步：它是一個放在你電腦上的資料夾，裡面寫好一份工作說明（SKILL.md），AI 代理就能讀取你指定資料夾裡的檔案、批次處理、再把結果檔案寫回電腦——例如一次批改整包作業、把整學期教材整理成計畫。設定一次，之後每次呼叫都照同一套流程做。

⚠️ Skill 需要桌面版的代理工具（能讀寫你電腦檔案的 Claude Code 或 OpenAI Codex），一般 ChatGPT 聊天、Gemini 的 Gem 做不到讀取你電腦的資料夾路徑。這區屬於進階，建議熟悉前面兩種類型後再嘗試，並先拿測試資料夾試跑、檢查產出無誤再用於正式資料。

怎麼開始（三步）

1安裝代理工具

在電腦安裝 Claude Code 或 OpenAI Codex（在終端機／桌面 App 執行，能存取本機檔案）。開一個資料夾當你的工作區。

2建立一個 skill

在 skill 資料夾裡建一個子資料夾（例如 grade-essays/），裡面放一個檔名固定為 SKILL.md 的檔案。

3寫內容並呼叫

SKILL.md 開頭用 --- name / description --- 標頭，下面寫步驟。之後在工具裡打 /skills 或用 $名稱 呼叫，或直接描述任務讓它自動判斷。

ℹ 放檔位置：Claude Code 用 ~/.claude/skills/、Codex 用 ~/.codex/skills/（放專案內則用 .claude/skills/／.codex/skills/）。SKILL.md 的格式兩邊共通，同一個資料夾可跨工具共用，只差放的位置不同。

四個常用範例

SKILL

整包作業批改

grade-essays

指向一個裝滿學生作文／作業的資料夾，依你的評分規準逐份批改，每份輸出一個回饋檔，最後彙整全班常見問題。

---
name: grade-essays
description: 依老師的評分規準，批改一個資料夾裡的學生作文，逐份輸出回饋。當老師要批改整批作業時使用。
---
# 批改作文
1. 讀取 essays/ 資料夾裡每一份 .docx 或 .txt。
2. 依 rubric.md 的評分規準逐份評分。
3. 每份輸出一個回饋檔到 feedback/：亮點、一個具體建議、鼓勵的話。
4. 最後彙整全班常見問題成 summary.md。
怎麼用：把作文放進 essays/、把你的規準寫成 rubric.md，然後說：「用 grade-essays 批改 essays 資料夾」。
SKILL

從課程講義出考卷

make-exam 豐富版示範

同樣是出考卷，這個版本把規範拆成 references/ 參考檔、加上品質確認清單，AI 出題更準、更一致。以下都是公開的 SKILL.md 通用寫法。

資料夾結構
make-exam/
├── SKILL.md
├── references/
│   ├── question-types.md   # 各題型的出題原則與配分
│   └── exam-format.md      # 考卷版面與作答說明範本
└── scripts/
    └── build_exam.py       # 選用：把考卷排版成 Word/PDF
SKILL.md
---
name: make-exam
description: 讀取課程講義資料夾，依指定範圍與題型產生考卷與解答。當老師要出考卷、練習卷或隨堂測驗時使用。
---
# 出考卷

## 開始前先讀
- references/question-types.md：各題型的出題原則與配分
- references/exam-format.md：考卷版面與作答說明範本

## 步驟
1. 讀取 materials/ 裡的講義（.pdf／.docx／.md），整理可出題的重點。
2. 向老師確認：範圍、總題數、題型比例、難易分布。
3. 依 references 的規範出題；題目只根據講義，不超綱。
4. 產出 exam.md 與 answer_key.md 到 output/。

## 品質確認清單
- [ ] 每題都能在講義中找到依據，未超綱
- [ ] 題型比例與題數符合指定
- [ ] 每題標明配分，總分加總正確
- [ ] 解答完整，附必要解題說明
- [ ] 版面依 exam-format.md，含作答說明與留白
怎麼用：把講義放進 materials/，說：「用 make-exam 出一份第 3-4 章、20 題的考卷」。
SKILL

學期資料夾整理成教學計畫

build-syllabus

掃描一學期散落的教案、投影片、講義，依週次或單元歸類，彙整成一份教學進度表，並標出缺漏。

---
name: build-syllabus
description: 掃描一學期散落的教案與講義資料夾，彙整成一份教學進度表／課程計畫。整理學期教學計畫時使用。
---
# 整理教學計畫
1. 掃描 semester/ 裡所有教案、投影片、講義。
2. 依週次或單元歸類，抓出每單元的目標與活動。
3. 產生 syllabus.md：週次、單元、學習目標、教學活動、評量。
4. 標出缺漏（例如某週沒有評量），提醒老師補。
怎麼用：把整學期檔案丟進 semester/，說：「用 build-syllabus 整理成 18 週進度表」。
SKILL

從教學資料做簡報

make-slides

讀取教案或講義，拆成一頁一頁的投影片內容，並在備註寫給老師的口說提示，可直接貼進簡報工具。

---
name: make-slides
description: 讀取教學資料，產生一份教學簡報大綱與內容。要把教材變成上課投影片時使用。
---
# 產生教學簡報
1. 讀取 lesson/ 裡的教案或講義。
2. 拆成投影片：標題、學習目標、每個重點一頁、活動頁、總結。
3. 產生 slides.md（每張以 --- 分頁），可貼進簡報工具或轉成 .pptx。
4. 每頁精簡條列，並在備註寫給老師的口說提示。
怎麼用：把教材放進 lesson/，說：「用 make-slides 做一份 12 頁的單元簡報」。

把 Skill 寫得更精準的 5 個技巧

前四個範例是精簡入門；想讓產出更穩定，往這五個方向補（都是公開的 SKILL.md 通用寫法）。

1
步驟寫具體、編號
把「做什麼」拆成 1-2-3，AI 少猜、少漏步驟。
2
用 references／ 放參考檔
把格式、規範、範本放成檔案讓它讀，它就有依據、不亂編。
3
需要固定格式就配 scripts／
每次都要同樣版面（如 .docx）時，用腳本產出更穩定。
4
結尾放品質確認清單
用 checklist 逼它自我檢查，交件品質更一致。
5
description 寫清楚「何時用」
加上觸發詞，工具才能在對的時機自動呼叫這個 skill。
